Fibonacci Numbers

Math behind why greedy works

Among all resolution with the minimum number of Fibonacci numbers, we are to find the lexicographically largest one.

Facts about shortest sequence:

  • uses each Fibonacci number at most once fibo[i] * 2 = fibo[i - 2] + fibo[i + 1]
  • never uses two consecutive Fibonacci numbers

Then: If no dup, no adjacent, we must take the biggest.

fibo[0] + fibo[2] + fibo[4] + ... + fibo[2n] = fibo[2n + 1] - 1

fibo[1] + fibo[3] + fibo[5] + .... + fibo[2n-1] = fibo[2n] - 1

Modular Inverse

A modular multiplicative inverse of an integer aaa is an integer xxx such that a⋅xa \cdot xa⋅x is congruent to 111 modular some modulus mmm. To write it in a formal way: we want to find an integer xxx so that

a⋅x≡1mod  m.a \cdot x \equiv 1 \mod m.a⋅x≡1modm.

We will also denote xxx simply with a−1a^{-1}a−1.

It can be proven that the modular inverse exists if and only if aaa and mmm are relatively prime (i.e. gcd⁡(a,m)=1\gcd(a, m) = 1gcd(a,m)=1).

Finding the Modular Inverse using Binary Exponentiation

  • For a prime modulus mmm: am−2≡a−1mod  ma ^ {m - 2} \equiv a ^ {-1} \mod mam−2≡a−1modm

Finding the Modular Inverse using Extended Euclidean algorithm

Use the identity eqaution (linear diophantine equation in two variables)

a⋅x+m⋅y=1a \cdot x + m \cdot y = 1a⋅x+m⋅y=1

Take modulu m both sides and you get the result.

The function below solves ax+by=1ax+by = 1ax+by=1, for given aaa and bbb.

def extended_euclidean(a, b):
    if b == 0:
        return 1, 0, a
    else:
        x, y, gcd = extended_euclidean(b, a % b)
        return y, x - (a // b) * y, gcd

Enumerating submasks of a bitmask

Given a bitmask mmm, you want to efficiently iterate through all of its submasks, that is, masks sss in which only bits that were included in mask mmm are set.

Consider the implementation of this algorithm, based on tricks with bit operations:

while sub:
    print(sub)
    sub = (sub-1) & m

The above code visits all submasks of mmm, without repetition, and in descending order. Suppose we have a current bitmask sss, and we want to move on to the next bitmask. By subtracting from the mask sss one unit, we will remove the rightmost set bit and all bits to the right of it will become 1. Then we remove all the "extra" one bits that are not included in the mask mmm and therefore can't be a part of a submask.

Iterating through all masks with their submasks. Complexity O(3n)O(3^n)O(3n)

In many problems, especially those that use bitmask dynamic programming, you want to iterate through all bitmasks and for each mask, iterate through all of its submasks:

for (int m=0; m<(1<<n); ++m):
    get_all_submasks(m)

Let's prove that the inner loop will execute a total of O(3n)O(3^n)O(3n) iterations.

First proof: Consider the iii-th bit. There are exactly three options for it:

  1. it is not included in the mask mmm (and therefore not included in submask sss),
  2. it is included in mmm, but not included in sss, or
  3. it is included in both mmm and sss.

As there are a total of nnn bits, there will be 3n3^n3n different combinations.

Second proof: Note that if mask mmm has kkk enabled bits, then it will have 2k2^k2k submasks. As we have a total of (nk)\binom{n}{k}(kn​) masks with kkk enabled bits (see binomial coefficients), then the total number of combinations for all masks will be:

∑k=0n(nk)⋅2k\sum_{k=0}^n \binom{n}{k} \cdot 2^k∑k=0n​(kn​)⋅2k

To calculate this number, note that the sum above is equal to the expansion of (1+2)n(1+2)^n(1+2)n using the binomial theorem. Therefore, we have 3n3^n3n combinations, as we wanted to prove.
